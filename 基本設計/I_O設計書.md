# I/O設計書

## 1. 文書目的

本書は、UML自動生成ツール初版における入力・出力・操作方式を定義するものである。

本書では以下を対象とする。

- GUI から受け付ける実行時入力
- `config.ini` に保持する固定寄り設定
- 内部 I/O の基本方針
- 出力成果物と保存方式

---

## 2. 適用範囲

### 2.1 初版前提

- 提供形態はローカル GUI アプリとする。
- 実行時に都度変更する項目は GUI から受け付ける。
- 固定寄り設定は `config.ini` で保持する。
- 正本出力は `.puml` とする。

### 2.2 初版対象外

- CLI 専用運用
- Web 版 UI
- 画像出力設定
- 詳細な除外ディレクトリ指定
- 複数起点関数指定
- 完全修飾名の手入力指定

---

## 3. 入力方式全体方針

### 3.1 GUI 優先

実行時に変わる主入力は GUI を正本とする。

対象:

- 解析対象ディレクトリ
- 出力先ディレクトリ
- 出力対象図
- 起点関数
- 型推論実施有無

### 3.2 `config.ini` の役割

`config.ini` は毎回変更しない固定寄り設定の保持に用いる。

### 3.3 内部受け渡し

- GUI が受け付けた値は `AnalysisRequest` に正規化して下流へ渡す。
- 起点関数選択後は `selected_entry_point_symbol_id` を付与して処理継続する。

---

## 4. GUI 入力項目

## 4.1 解析実行前入力

| 項目 | 必須 | 入力方式 | 説明 |
|---|---:|---|---|
| 解析対象ディレクトリ | ○ | GUI 選択 | プロジェクトの親ディレクトリ |
| 出力先ディレクトリ | ○ | GUI 選択 | `.puml` 保存先 |
| 出力対象図 | ○ | GUI チェックボックス等 | クラス図 / モジュール依存図 / シーケンス図 |
| 型推論実施有無 | ○ | GUI チェックボックス | 推論を行うか |

## 4.2 起点関数選択入力

シーケンス図を生成する場合、または型推論精度向上のため必要な場合、以下を GUI で受け付ける。

| 項目 | 必須 | 入力方式 | 説明 |
|---|---:|---|---|
| 起点関数候補選択 | 条件付き必須 | 一覧選択 | 候補一覧から 1 件選択 |

### 候補表示項目

- モジュールパス
- クラス名
- 関数名
- 引数数
- 行番号

### 内部保持

- GUI 表示名ではなく `symbol_id` を保持する。

---

## 5. GUI 画面単位の I/O

## 5.1 入力フォーム

### 入力

- ユーザ指定の解析対象ディレクトリ
- ユーザ指定の出力先ディレクトリ
- 出力対象図
- 型推論実施有無

### 出力

- `AnalysisRequest`

## 5.2 起点関数選択画面

### 入力

- `EntryPointCandidateSet`

### 出力

- 選択済み `symbol_id`

## 5.3 プレビュー画面

### 入力

- `GeneratedDiagramSet`

### 出力

- PlantUML テキスト表示
- 保存操作イベント

## 6. `config.ini` 設計

## 6.1 基本方針

- 実行ごとに変わる値は原則 GUI で指定する。
- `config.ini` は補助的な既定値・固定値を保持する。
- GUI 指定値がある場合は GUI を優先する。

## 6.2 想定保持項目

### `[app]`

| 項目 | 説明 |
|---|---|
| `default_output_dir` | 既定の出力先 |
| `last_open_dir` | 最後に開いた解析対象ディレクトリ |
| `default_diagram_types` | 初期選択図種 |

### `[preview]`

| 項目 | 説明 |
|---|---|
| `font_size` | プレビュー表示用フォントサイズ |
| `wrap_text` | テキスト折返し有無 |

### `[analysis]`

| 項目 | 説明 |
|---|---|
| `default_enable_type_inference` | 推論の既定値 |
| `loop_trace_limit` | ループ追跡回数の既定上限 |
| `recursion_note_enabled` | 再帰検知 note 出力有無 |

### `[output]`

| 項目 | 説明 |
|---|---|
| `class_diagram_filename` | クラス図出力ファイル名既定値 |
| `module_diagram_filename` | 依存図出力ファイル名既定値 |
| `sequence_diagram_prefix` | シーケンス図ファイル名接頭辞 |

## 6.3 優先順位

設定値の優先順位は以下とする。

1. GUI 明示指定
2. `config.ini`
3. アプリ内部デフォルト

---

## 7. 内部入力データ

## 7.1 `AnalysisRequest`

GUI から下流へ渡す正規入力。

| 項目 | 型 | 必須 | 説明 |
|---|---|---:|---|
| `request_id` | UUID | ○ | 実行要求識別子 |
| `target_root_path` | str | ○ | 解析対象ルート |
| `output_dir` | str | ○ | 出力先 |
| `diagram_types` | set[str] | ○ | 対象図種 |
| `enable_type_inference` | bool | ○ | 推論有無 |
| `selected_entry_point_symbol_id` | UUID \| None | △ | 起点関数 |
| `config_snapshot` | AnalysisOptionSnapshot | △ | 実行時設定のスナップショット |

## 7.2 `AnalysisOptionSnapshot`

実行時点で有効だった設定値群を記録する。

保持例:

- GUI で選択された解析対象ディレクトリ
- GUI で選択された出力先
- GUI で選択された出力対象図
- 型推論実施有無
- `config.ini` の既定値反映結果

---

## 8. 出力成果物

## 8.1 ファイル出力

| 成果物 | 必須 | 形式 | 説明 |
|---|---:|---|---|
| クラス図 | 条件付き | `.puml` | クラス図 PlantUML |
| モジュール依存図 | 条件付き | `.puml` | 依存図 PlantUML |
| シーケンス図 | 条件付き | `.puml` | 起点関数ベースのシーケンス図 |

### 条件付きの意味

- GUI で選択された図のみ出力する。

## 8.2 GUI 出力

| 成果物 | 形式 | 説明 |
|---|---|---|
| PlantUML プレビュー | テキスト | GUI 内で内容確認するための表示 |
| 起点関数候補一覧 | リスト | 候補選択用表示 |
| Diagnostic 表示 | テキスト / リスト | 警告・情報の利用者向け表示 |

---

## 9. ファイル命名規則

### 9.1 基本命名

- クラス図: `class_diagram.puml`
- モジュール依存図: `module_dependency_diagram.puml`
- シーケンス図: `sequence_diagram_<entrypoint>.puml`

### 9.2 `<entrypoint>` の生成

- 完全修飾名そのままは使わない。
- ファイル名に使えない文字は置換する。
- 衝突回避のため短縮 `symbol_id` を補助付与してよい。

例:

- `sequence_diagram_pkg_sample_main_a1b2c3.puml`

---

## 10. 入力値検証ルール

## 10.1 解析対象ディレクトリ

- 実在するディレクトリであること
- 読み取り可能であること

## 10.2 出力先ディレクトリ

- 実在する、または作成可能であること
- 書き込み可能であること

## 10.3 出力対象図

- 少なくとも 1 種は選択されていること

## 10.4 起点関数

- シーケンス図選択時は必須とする
- 選択済み `symbol_id` が `ProjectModel` 内 callable と整合すること

---

## 11. 異常時の I/O 方針

## 11.1 GUI 入力異常

- 不正入力は実行前に GUI 上で通知する。
- 可能なものは GUI 側で事前検証する。

## 11.2 解析中異常

- 解析中の警告・情報は Diagnostic として蓄積する。
- GUI は実行後に一覧表示できるようにする。

## 11.3 保存異常

- 保存失敗時は GUI にエラー表示する。
- 解析済み PlantUML テキストは可能ならプレビューに残す。

---

## 12. 操作シナリオ

## 12.1 クラス図 / 依存図のみ生成

1. ユーザが解析対象ディレクトリを選択する。
2. ユーザが出力先を選択する。
3. ユーザがクラス図 / モジュール依存図を選択する。
4. 必要に応じて型推論有無を指定する。
5. 実行する。
6. `.puml` を保存し、GUI に表示する。

## 12.2 シーケンス図生成

1. 上記実行前入力を指定する。
2. 中間モデル生成後、起点関数候補一覧を表示する。
3. ユーザが起点関数を 1 件選択する。
4. 推論・追跡を実行する。
5. シーケンス図 `.puml` を保存し、GUI に表示する。

## 14. 決定事項まとめ

- 実行時に変わる主要入力は GUI を正本とする。
- 固定寄り設定は `config.ini` で保持する。
- GUI 入力は `AnalysisRequest` に正規化して下流へ渡す。
- シーケンス図生成時は起点関数候補一覧から 1 件選択する。
- 起点関数の内部保持は `symbol_id` とする。
- 正本出力は `.puml` とする。
- GUI では PlantUML テキストのプレビューを提供する。
- 設定値優先順位は GUI > `config.ini` > 内部デフォルトとする。
