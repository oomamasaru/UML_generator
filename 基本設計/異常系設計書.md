# 異常系設計書

## 1. 文書目的

本書は、UML自動生成ツール初版における異常系・診断情報・継続可否の判断基準を定義するものである。

本書の対象は以下とする。

- Diagnostic の体系
- エラー・警告・情報の分類
- レイヤーごとの異常検出ポイント
- 継続 / 中断基準
- 利用者向け表示と開発者向け記録の分離方針

---

## 2. 基本方針

### 2.1 継続優先

- 一部失敗が発生しても、可能な範囲で解析全体を継続する。
- 型不明、参照未解決、外部依存、未対応構文は原則として継続対象とする。

### 2.2 誤解決禁止

- 情報が足りない場合に、もっともらしいが根拠のない結果を確定しない。
- 不明は `Unknown`、未解決は unresolved として保持する。

### 2.3 診断と結果の分離

- 診断情報は `Diagnostic` として独立保持する。
- 成果物は可能な範囲で生成し、異常は診断側で伝える。

---

## 3. Diagnostic モデル

## 3.1 基本構造

`Diagnostic` は以下を保持する。

- `diagnostic_id`
- `level`
- `code`
- `message`
- `span_id`
- `related_symbol_id`
- `related_ref_id`

## 3.2 レベル定義

| level | 意味 | 典型例 |
|---|---|---|
| `ERROR` | 当該単位の継続困難な失敗 | parse failed, output dir invalid |
| `WARNING` | 継続可能だが精度・完全性に影響 | unresolved symbol, unknown type |
| `INFO` | 未対応構文記録や補足情報 | skipped unsupported syntax |

## 3.3 コード体系

コードは文字列とし、以下のプレフィックスを用いる。

- `PARSE`
- `RESOLVE`
- `MODEL`
- `INFER`
- `GENERATE`
- `CONFIG`
- `GUI`

例:

- `PARSE001`
- `RESOLVE101`
- `INFER201`

---

## 4. UnsupportedConstructRecord 方針

### 4.1 目的

初版対象外構文を黙って捨てず、後続改善可能な形で記録する。

### 4.2 記録対象

- list 内包表記
- dict / set 内包表記
- 三項演算子
- `async` / `await`
- `yield`
- context manager の詳細展開対象

### 4.3 記録内容

- 構文種別
- ソース位置
- 対応レイヤー
- スキップ理由
- 関連 callable / source unit

### 4.4 レベル

- 原則 `INFO` とする。
- ただし当該構文が解析全体を破壊する場合のみ `WARNING` 以上へ引き上げる。

## 5. レイヤー別異常一覧

## 5.1 GUI Layer

| 条件 | level | code プレフィックス | 継続可否 | 対応 |
|---|---|---|---|---|
| 解析対象未指定 | `ERROR` | `GUI` | 中断 | 入力要求 |
| 出力先未指定 | `ERROR` | `GUI` | 中断 | 入力要求 |
| 図種未選択 | `ERROR` | `GUI` | 中断 | 入力要求 |
| シーケンス図なのに起点未選択 | `ERROR` | `GUI` | 中断 | 起点選択要求 |

## 5.2 File Collection Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| 対象ディレクトリ不存在 | `ERROR` | `CONFIG001` | 中断 | 実行停止 |
| 対象ディレクトリ読取不可 | `ERROR` | `CONFIG002` | 中断 | 実行停止 |
| 一部ファイル読取不可 | `WARNING` | `CONFIG101` | 継続 | 読めたファイルのみ継続 |
| Python ファイル 0 件 | `WARNING` | `CONFIG102` | 継続 | 空結果扱い |

## 5.3 Parser Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| 単一ファイル parse 失敗 | `ERROR` | `PARSE001` | 継続 | 当該 file をスキップ |
| 未対応構文検出 | `INFO` | `PARSE101` | 継続 | UnsupportedConstructRecord 記録 |
| 型ヒント未記載 | `INFO` | `PARSE102` | 継続 | `Unknown` 扱い |

## 5.4 Symbol Resolution Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| import 未解決 | `WARNING` | `RESOLVE001` | 継続 | unresolved として保持 |
| symbol 未解決 | `WARNING` | `RESOLVE002` | 継続 | candidate 0 件許容 |
| 複数候補競合 | `WARNING` | `RESOLVE003` | 継続 | confidence / rank 付き候補保持 |
| 相対 import 正規化失敗 | `WARNING` | `RESOLVE004` | 継続 | unresolved 扱い |

## 5.5 Intermediate Model Generation Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| payload 構築失敗 | `ERROR` | `MODEL001` | 条件付き継続 | 当該 symbol を部分スキップ |
| behavior graph 構築失敗 | `WARNING` | `MODEL002` | 継続 | 構造情報のみ残す |
| property 統合失敗 | `WARNING` | `MODEL003` | 継続 | callable 単位で残す |

## 5.6 Type Inference Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| 起点関数未指定 | `ERROR` | `INFER001` | 中断 | シーケンス生成不可 |
| 型推論不能 | `WARNING` | `INFER101` | 継続 | `Unknown` 維持 |
| ループ追跡上限到達 | `INFO` | `INFER102` | 継続 | 構造保持のまま打切り |
| 再帰 / 循環検知 | `INFO` | `INFER103` | 継続 | note 用情報を残して追跡停止 |

## 5.7 Diagram Generator Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| `ProjectModel` 不正 | `ERROR` | `GENERATE001` | 中断 | 生成失敗 |
| class 図対象 0 件 | `WARNING` | `GENERATE101` | 継続 | 空図または未生成 |
| module 依存 0 件 | `WARNING` | `GENERATE102` | 継続 | 空図または未生成 |
| unresolved 多数 | `WARNING` | `GENERATE103` | 継続 | 部分図を生成 |

## 5.8 Preview / Export Layer

| 条件 | level | code | 継続可否 | 対応 |
|---|---|---|---|---|
| 出力先書込不可 | `ERROR` | `GENERATE002` | 条件付き | プレビューのみ継続 |
| ファイル保存失敗 | `ERROR` | `GENERATE003` | 条件付き | GUI に通知 |

## 6. 継続 / 中断基準

## 6.1 中断条件

以下は要求全体を中断する。

- 解析対象ディレクトリが不正
- 出力先が不正で、かつ代替出力もできない
- シーケンス図必須なのに起点関数が選択できない
- `ProjectModel` 自体を構築できない

## 6.2 継続条件

以下は原則継続する。

- 一部ファイルの parse 失敗
- 一部 import 未解決
- symbol 未解決
- 型不明
- 未対応構文
- 外部ライブラリ参照
- ループ追跡打切り
- 再帰 / 循環検知

---

## 7. 利用者向け表示方針

## 7.1 GUI 表示対象

利用者向けには以下を見やすく表示する。

- level
- message
- 対象ファイル
- 行番号

## 7.2 表示粒度

- `ERROR` は目立つ形で表示する。
- `WARNING` は結果精度へ影響する旨を示す。
- `INFO` は折りたたみ可能でもよい。

## 7.3 表示メッセージ方針

- 実装内部名だけで済ませない。
- 何が起きたか、結果へどんな影響があるかを短く示す。

例:

- `sample.py:15 の import を解決できなかったため、この依存関係は図に含まれません。`

---

## 8. 開発者向け記録方針

## 8.1 詳細情報

開発者向けには以下を保持してよい。

- raw な code
- related symbol / ref
- span
- confidence
- rank
- 例外メッセージ要約

## 8.2 ログとの分離

- `Diagnostic` はユーザに見せる前提の構造化データとする。
- 生の例外トレースは必要に応じて開発ログへ分離する。

---

## 9. 代表 Diagnostic 一覧

| code | level | 内容 |
|---|---|---|
| `CONFIG001` | `ERROR` | 解析対象ディレクトリが存在しない |
| `CONFIG002` | `ERROR` | 解析対象ディレクトリを読めない |
| `CONFIG101` | `WARNING` | 一部ファイルを読めない |
| `PARSE001` | `ERROR` | ファイルの AST 生成に失敗 |
| `PARSE101` | `INFO` | 未対応構文をスキップした |
| `RESOLVE001` | `WARNING` | import を解決できない |
| `RESOLVE002` | `WARNING` | symbol を解決できない |
| `MODEL002` | `WARNING` | Behavior IR の一部を構築できない |
| `INFER001` | `ERROR` | 起点関数未指定で推論できない |
| `INFER102` | `INFO` | ループ追跡上限に到達 |
| `INFER103` | `INFO` | 再帰 / 循環を検知して追跡停止 |
| `GENERATE001` | `ERROR` | 図生成に必要な中間モデルが不正 |
| `GENERATE003` | `ERROR` | ファイル保存に失敗 |

---

## 10. 決定事項まとめ

- 継続可能な失敗は原則として処理継続する。
- 不明な情報は誤って確定せず、Unknown / unresolved で保持する。
- 異常は `Diagnostic` として構造化して保持する。
- `ERROR` / `WARNING` / `INFO` の 3 段階で扱う。
- 未対応構文は `UnsupportedConstructRecord` と `Diagnostic(INFO)` で残す。
- GUI 向け表示と開発者向け詳細情報は役割分離する。
