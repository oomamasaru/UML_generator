# 出力仕様書

## 1. 文書目的

本書は、UML自動生成ツール初版における図出力仕様を定義するものである。

本書で定義する対象は以下の通りとする。

- 出力対象図
- 図ごとの表示対象
- Unknown / unresolved / Union の表示方針
- PlantUML 生成ルール
- ファイル出力ルール

---

## 2. 適用範囲

### 2.1 初版対象図

初版で正式対応する図は以下の 3 種とする。

- クラス図
- モジュール依存図
- シーケンス図

### 2.2 初版出力記法

- 初版の出力記法は PlantUML のみとする。
- GUI 上では PlantUML テキストをプレビュー可能とする。
- 正本成果物は `.puml` ファイルとする。

### 2.3 初版対象外

以下は将来対応とする。

- Mermaid
- Graphviz DOT
- XMI
- PNG / SVG 等の画像出力
- コンポーネント図
- アクティビティ図
- 状態遷移図
- 通信図

---

## 3. 共通出力方針

### 3.1 入力モデル

- Diagram Generator Layer は `ProjectModel` のみを参照する。
- AST、Parser 出力、Collector 出力、Resolver 出力には戻らない。
- 推論未実施の場合は、推論結果が空の `ProjectModel` をそのまま利用する。

### 3.2 欠損情報の扱い

- 型不明は `Unknown` として表示可能にする。
- unresolved は出力不能理由にはせず、可能な範囲で図生成を継続する。
- 一部の関係が欠けても、図全体の生成は継続する。

### 3.3 名称管理

- 解決用名称は `canonical_name` を用いる。
- 図表示名は Generator が派生生成する。
- 表示名は原則として最短一意名を採用する。
- 同名衝突時は右側から 1 階層ずつ広げる。

例:

- `service.user.User` → `User`
- `domain.user.User` と衝突する場合 → `user.User`
- さらに衝突する場合 → `service.user.User`

### 3.4 外部ライブラリ

- 初版では外部ライブラリは図上非表示とする。
- 外部呼び出し実績は Diagnostic や将来拡張用補助情報には保持してよい。

### 3.5 ファイル出力

- 図 1 種ごとに 1 ファイルを出力する。
- エンコーディングは UTF-8 とする。
- 改行コードは LF を標準とする。

---

## 4. クラス図仕様

### 4.1 目的

クラス図は、型宣言およびその構造関係を表現することを目的とする。

### 4.2 表示対象

- `TypeDecl(CLASS)`
- `FieldDecl`
- `PropertyDecl`
- `CallableDecl`
- 継承関係
- 保持関係
- 利用関係

### 4.3 クラス本体の表示要素

各クラスには必要に応じて以下を出力する。

- クラス名
- フィールド一覧
- property 一覧
- メソッド一覧

### 4.4 フィールド表示

- `FieldDecl` を表示対象とする。
- 型が declared type で確定している場合は declared type を表示する。
- declared type が不明で推論結果がある場合は、推論結果を補助表示してよい。
- どちらも不明な場合は `Unknown` を表示する。

表示例:

```text
sample2: Sample2
arg1: Unknown
```

### 4.5 property 表示

- `PropertyDecl` を通常の属性相当として表示する。
- getter / setter の有無は必要に応じて修飾情報として付与してよい。

### 4.6 メソッド表示

- `CallableDecl` のうち class 配下の callable を表示する。
- 引数型、戻り値型は可能な範囲で表示する。
- 型不明は `Unknown` を表示する。

### 4.7 継承関係

- `Relation(EXTENDS)` を用いて継承矢印を出力する。
- 複数継承は複数の継承矢印として表現する。

### 4.8 保持関係

- `Relation(HAS_A)` を用いて保持関係を表現する。
- 型が unresolved の場合は関係線を出さず、属性側に `Unknown` を残すことを優先する。

### 4.9 利用関係

- `Relation(USES)` は必要に応じて表示する。
- 初版では図の可読性を優先し、常時表示を必須とはしない。

### 4.10 PlantUML 出力ルール

- class は `class <表示名> { ... }` 形式で出力する。
- 継承は `<親> <|-- <子>` 形式で出力する。
- 保持は `<owner> --> <type>` 等の表記で表現してよい。

---

## 5. モジュール依存図仕様

### 5.1 目的

モジュール依存図は、プロジェクト内モジュール間の import 依存関係を可視化することを目的とする。

### 5.2 表示対象

- `SourceUnit`
- `Relation(IMPORTS)`

### 5.3 ノード単位

- 初版ではファイル / モジュール単位でノードを表現する。
- package 情報は module path に反映する。

### 5.4 依存関係

- `import`
- `from ... import ...`
- alias import
- relative import

上記は最終的にモジュール間依存へ正規化する。

### 5.5 外部依存

- プロジェクト外モジュールは図上非表示とする。
- そのため、`import os` などの標準ライブラリは図に出さない。

### 5.6 PlantUML 出力ルール

- モジュールノードは `package` または `class` 相当の簡易表現でよい。
- 依存矢印は `-->` を基本とする。

---

## 6. シーケンス図仕様

### 6.1 目的

シーケンス図は、起点関数からの呼び出し追跡結果を可視化することを目的とする。

### 6.2 生成条件

- シーケンス図生成時は起点関数指定を必須とする。
- 起点関数は GUI の候補一覧から 1 件選択する。
- 推論・追跡を経て `CallTraceStep` または等価な派生結果が得られていることを前提とする。

### 6.3 participant 単位

- 初版ではクラス単位で participant を表現する。
- インスタンス単位表現は将来対応とする。

### 6.4 participant 表示名

- 原則として完全修飾名の最終要素を採用する。
- 衝突時は右から 1 階層ずつ広げる。

### 6.5 表示対象

- プロジェクト内で解決可能な call / create
- 分岐 (`if`) に基づく alt 表現
- ループ (`for` / `while`) に基づく loop 表現
- return
- 再帰 / 循環検知 note

### 6.6 call / create 表示

- `BehaviorNode(CALL)` はメッセージ呼び出しとして表示する。
- `BehaviorNode(CREATE)` は生成メッセージとして表示する。
- クラス生成時の生成先 participant は生成対象 class とする。

### 6.7 分岐表示

- `if` は `alt` / `else` により表現する。
- guard_text がある場合は分岐条件として表示する。
- 型条件分岐も同様に扱う。

### 6.8 ループ表示

- `for` / `while` は `loop` で表現する。
- guard_text がある場合は loop ラベルとして表示してよい。
- 初版では経路爆発防止のため、追跡回数制御が掛かっていても構造は loop として表現する。

### 6.9 return 表示

- return は原則として戻りメッセージとして出力する。
- 戻り値詳細は冗長になる場合、省略可能とする。

### 6.10 再帰・循環呼び出し

- 追跡中スタックに同一 callable が存在する場合は再帰 / 循環とみなす。
- 検知時は note を表示する。
- 当該呼び出し以降の deeper trace は中断する。

### 6.11 外部呼び出し

- 初版ではプロジェクト外呼び出しは表示しない。
- participant のみ表示して call / return を省略する方式は採用しない。

### 6.12 PlantUML 出力ルール

- participant は `participant <表示名>` を基本とする。
- call は `A -> B: method()` を基本とする。
- return は `B --> A` を基本とする。
- 分岐は `alt` / `else` / `end` を使用する。
- ループは `loop` / `end` を使用する。
- 再帰 / 循環検知は `note over` で表現する。

---

## 7. Unknown / unresolved / Union 表示方針

### 7.1 Unknown

- 表示可能な場面では `Unknown` と明示する。
- 黙って空欄にはしない。

### 7.2 unresolved

- unresolved な参照は、無理に誤った対象へ結び付けない。
- 関係線を出さない、またはテキスト上で不明とする。

### 7.3 Union

- 必要に応じて `A | B` のように表示する。
- シーケンス図では分岐追跡結果がある場合、`alt` へ展開して表現してよい。

---

## 8. ファイル命名規則

### 8.1 出力ファイル名

初版では以下を基本とする。

- クラス図: `class_diagram.puml`
- モジュール依存図: `module_dependency_diagram.puml`
- シーケンス図: `sequence_diagram_<entrypoint>.puml`

### 8.2 シーケンス図の起点識別

- `<entrypoint>` は表示名ではなく、安全なファイル名へ変換した識別子を用いる。
- 必要に応じて `symbol_id` の短縮表現を併用して衝突を防ぐ。

---

## 9. 生成不可条件

以下は当該図の生成不可条件とする。

### 9.1 クラス図

- `ProjectModel` 自体が未生成

### 9.2 モジュール依存図

- `SourceUnit` が 0 件

### 9.3 シーケンス図

- 起点関数未選択
- 起点関数に対応する callable が中間モデル上に存在しない

補足:

- 一部の call 解決失敗や型不明は生成不可条件としない。

## 11. 決定事項まとめ

- 初版対象図はクラス図、モジュール依存図、シーケンス図の 3 種とする。
- 初版出力記法は PlantUML のみとする。
- 正本成果物は `.puml` とする。
- Generator は `ProjectModel` のみを参照する。
- Unknown は明示表示する。
- unresolved は誤結線せず、可能な範囲で図生成を継続する。
- 外部ライブラリは初版では図上非表示とする。
- シーケンス図はクラス単位 participant とする。
- 分岐は `alt`、ループは `loop`、再帰 / 循環は note で表現する。
